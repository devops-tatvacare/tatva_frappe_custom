title: "Discovery — introspect, don't hardcode"
---

These docs deliberately do **not** dump a master field list. The fields you may send and read are **per-partner** (driven by your Allowed Fields grid) and they **evolve** — new verticals add new profile fields over time. So the source of truth is the API itself, at runtime, for **your** key.

## `lead_schema` is your per-partner source of truth

Call `lead_schema` first in any integration. It returns **exactly** the parent + child fields THIS key may send/read — with each field's `fieldname`, `label`, `type`, `required`, and `options` — plus the child-write rules, the dedup grain, the bulk limits, and your routing mode:

```bash
curl -H 'Authorization: token <api_key>:<api_secret>' \
     https://one.tatvacare.in/api/method/tatva_connect.api.partner.lead_schema
```

Two partners hitting this get **different** field lists. Read it; don't hardcode a list copied from a doc — a doc can't know what your key is allowed to do.

What to read out of `data`:

- **`data.lead[]`** — allowed parent (CRM Lead) fields.
- **`data.children.<table>`** — per child table: `multi_row` (bool), `key_field` (the required key on a multi-row child — see [Child tables](/docs/frappe-child-tables)), and `fields[]`.
- **`data.routing.mode`** — `forced` (partner; line fixed) or `caller-supplied` (internal key sends routing).
- **`data.dedup`** / **`data.bulk`** — the lead grain and the bulk/list limits.

## Going deeper: native DocType introspection

For values behind a `Link`/`Select` (e.g. which `CRM Program` rows exist), a DocType's own definition is itself a document, so you read its schema over the same REST API:

```
GET /api/resource/DocType/CRM Lead
```

You get every field with its `fieldname`, `label`, `fieldtype`, and — for `Select`/`Link` fields — its allowed `options`. (This is a System-Manager-only call.)

**The pattern to build:** *read the schema (`lead_schema` for your contract; `DocType` for link values) → map your data → write.* When we add a field or grant you a new one, your code adapts instead of breaking.

**Example:** before pushing leads with a new `custom_current_program`, introspect `CRM Program` to confirm the program exists rather than hardcoding a name that may have been renamed.
