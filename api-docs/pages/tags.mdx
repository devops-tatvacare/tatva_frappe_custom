---
title: Tags
---

Tags are free-form labels on a lead — `vip`, `nps-detractor`, `reactivation`, whatever the team needs. They're the fastest way to slice leads without adding a field, and an integrator can both **set** tags and **find** leads by tag over the API.

## How tags are stored

Frappe keeps tags in two places and keeps them in sync:

- A denormalized **`_user_tags`** string on the lead itself (comma-joined) — handy for a quick `like` filter.
- A **`Tag Link`** row per (tag, document) pair — the clean way to ask "which leads carry this tag?"

The tag vocabulary itself lives in the **`Tag`** doctype, one row per distinct label.

## Setting and removing

Never write `_user_tags` by hand — you'd desync the two stores. Use the two whitelisted methods, which update both:

- **Add** — `add_tag(tag, dt, dn)` tags document `dn` of doctype `dt`.
- **Remove** — `remove_tag(tag, dt, dn)`.

## Quick check — find leads by tag

- **All tags in use:** list the `Tag` doctype.
- **Leads carrying a tag:** filter `Tag Link` by `tag` (it returns the linked lead names), or filter the lead list with `_user_tags like %vip%`.
