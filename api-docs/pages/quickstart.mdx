## Four calls to your first lead

This walks you through the minimum to go from zero to a verified lead in production. Run each call sequentially — copy the response of one into the next.

### 1. Verify the key works

```bash
curl -H 'Authorization: token <api_key>:<api_secret>' \
     https://one.tatvacare.in/api/method/frappe.auth.get_logged_user
```
Expected: `{"message":"<your-integration-user>@tatvacare.in"}`. If you get HTML or a 401, the auth header is wrong — see [Authentication](/docs/api/authentication).

### 2. Get the valid routing values

Routing fields on `CRM Lead` (`custom_vertical`, `custom_psp_group`, `custom_current_program`) are Links to small lookup tables. Read them first so you know what to send:

```bash
curl -H 'Authorization: token KEY:SECRET' \
     https://one.tatvacare.in/api/resource/CRM%20Vertical
# → {"data":[{"name":"GoodFlip"},{"name":"GoodFlip Care"},{"name":"Pill Up"},{"name":"TatvaPractice"}]}
```

Repeat for `CRM Group` and `CRM Program`. See [Lookups](/docs/api/reference-data).

### 3. Create your first lead (minimum payload)

The [Upsert](/docs/api/leads) endpoint is the recommended write — idempotent on `mobile_no`, safe to retry.

```bash
curl -X POST -H 'Authorization: token KEY:SECRET' -H 'Content-Type: application/json' \
     -d '{"mobile_no":"+919999000099","first_name":"Test Lead","status":"New"}' \
     https://one.tatvacare.in/api/method/upsert_lead_by_phone
```

Expected response — note the `name` returned, that's the CRM's primary key for this lead:

```json
{"message":{"ok":true,"action":"created","name":"CRM-LEAD-2026-NNNNN","mobile_no":"+919999000099"}}
```

**Always check `message.ok`** (boolean), NOT the HTTP status. The endpoint returns HTTP 200 even on failure — a known quirk pending patch.

### 4. Read it back

```bash
curl -H 'Authorization: token KEY:SECRET' \
     'https://one.tatvacare.in/api/resource/CRM%20Lead/CRM-LEAD-2026-NNNNN'
```

Returns the full lead doc with embedded `custom_plan_profile[]`, `custom_lab_profile[]`, `custom_care_providers_profile[]` arrays. They'll be empty since you didn't send any child rows.
