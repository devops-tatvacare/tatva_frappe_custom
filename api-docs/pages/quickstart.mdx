The minimum to go from zero to a verified lead in production. Run each call sequentially — copy the response of one into the next.

## Four calls to your first lead

### 1. Verify the key works

```bash
curl -H 'Authorization: token <api_key>:<api_secret>' \
     https://one.tatvacare.in/api/method/frappe.auth.get_logged_user
```
Expected: `{"message":"<your-integration-user-email>"}` (this is a *native* Frappe method, so it uses the `message` wrapper). If you get HTML or a 401, the auth header is wrong — see [Authentication](/api/authentication).

### 2. Discover what you may send

Call `lead_schema` to get exactly the fields YOUR key may send/read and your routing mode (`forced` if you're a partner, `caller-supplied` if you're an internal key):

```bash
curl -H 'Authorization: token <api_key>:<api_secret>' \
     https://one.tatvacare.in/api/method/tatva_connect.api.partner.lead_schema
```

If you're a **partner**, your routing is fixed — skip straight to step 3. If you're an **internal** key, you supply routing (`custom_vertical`, `custom_group`, `custom_current_program`); read their valid values from the lookup tables first:

```bash
curl -H 'Authorization: token <api_key>:<api_secret>' \
     https://one.tatvacare.in/api/resource/CRM%20Vertical
# → {"data":[{"name":"Product Line B"},{"name":"Product Line A"},{"name":"Pill Up"},{"name":"TatvaPractice"}]}
```

Repeat for `CRM Group` and `CRM Program`. See [Lookups](/api/reference-data).

### 3. Create your first lead (minimum payload)

`lead_create` is the default write — idempotent on the lead grain (phone + line + group), safe to retry.

```bash
curl -X POST -H 'Authorization: token <api_key>:<api_secret>' -H 'Content-Type: application/json' \
     -d '{"mobile_no":"+919900000000","first_name":"Test Lead","status":"New"}' \
     https://one.tatvacare.in/api/method/tatva_connect.api.partner.lead_create
```

Expected response — **top-level** `status`/`action`/`data` (the partner methods do **not** wrap the payload in `message`). Note `data.name`, the CRM's primary key for this lead (PUT it back to `lead_update`):

```json
{
  "status": "success",
  "action": "created",
  "data": {
    "name": "CRM-LEAD-2026-NNNNN",
    "source": "Program D",
    "vertical": "Product Line B",
    "group": "Group B",
    "program": "Program D"
  }
}
```

A validation or permission failure returns the unified error contract — `{"status":"error","error":{"code":…,"message":…}}` with a matching HTTP status. See [Error contracts](/errors) and [Reading responses](/reading-values).

### 4. Read it back

```bash
curl -H 'Authorization: token <api_key>:<api_secret>' \
     'https://one.tatvacare.in/api/method/tatva_connect.api.partner.lead_get?name=CRM-LEAD-2026-NNNNN'
```

Returns the lead with your allowed fields plus embedded `custom_plan_profile[]` / `custom_lab_profile[]` arrays. They'll be empty since you didn't send any child rows.
