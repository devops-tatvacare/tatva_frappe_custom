Reference for integrating with the TatvaCare CRM — the single source of truth for leads across every TatvaCare product line. It's the systems door into the CRM: external systems push leads in and read state back, while internal teams work those leads in the CRM web app.

## What this API does

- **Write leads** — create or update by phone (idempotent), single or bulk.
- **Read leads** — fetch one, list with filters, or count.
- **Discover the data model** — introspect any doctype's schema at runtime.
- **Attach files** — upload documents against a lead.

## Who it's for

- Engineers wiring an **inbound lead feed** into the CRM (e.g. an insurer sending lab-screened leads).
- Engineers wiring a **parallel write** from a system that already creates leads elsewhere.
- Anyone who needs to **read** leads, their profiles, or their substages from outside the CRM.

## The shape

```
 CRM Lead (one lead per person × product line + group)
   ├── Plan Profile           — what plan / PSP / insurance / drug program / subscription
   ├── Lab Profile            — lab values, vitals, conditions, scores
   └── Care & Providers       — coaches, RM, therapist, doctor, hospital
```

The lead carries identity, contact, and routing fields. The three children carry concern-specific data. A lead typically has 1–3 child rows populated; multi-program is normal.

**Lead grain:** a lead is unique per `mobile_no` + `custom_vertical` (product line) + `custom_group`. The same phone on a different line or group is a separate lead — one person can own several leads. Re-sending the same trio updates that one lead; it never duplicates.

## Scope today

This API focuses on **Lead**, the record every integrator touches. The CRM has other doctypes too — Tasks, Notes, Call Logs, and more — all reachable through the same [REST pattern](/frappe-rest-api). We document the surface as integrations need it: if you need one that isn't covered, introspect it live with [Runtime discovery](/frappe-discovery) or ask CRM Ops.

If a field, value, or behavior isn't documented here, don't guess — use [Runtime discovery](/frappe-discovery) to introspect the live model, or email CRM Ops.
