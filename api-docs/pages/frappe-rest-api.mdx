---
title: "The auto-generated REST API"
---

Because every record is a DocType, Frappe exposes **every DocType over REST automatically** — no endpoint is hand-written for CRUD. Two URL shapes cover almost everything.

**1. Resource API — `/api/resource/<DocType>`** (URL-encode the space, e.g. `CRM%20Lead`):

| Method | URL | Does |
|---|---|---|
| `GET` | `/api/resource/CRM Lead` | list (filter + paginate) |
| `GET` | `/api/resource/CRM Lead/{name}` | read one |
| `POST` | `/api/resource/CRM Lead` | create |
| `PUT` | `/api/resource/CRM Lead/{name}` | update |
| `DELETE` | `/api/resource/CRM Lead/{name}` | delete |

**2. Method API — `/api/method/<dotted.path>`** for custom server functions — e.g. `upsert_lead_by_phone`, which finds-or-creates a lead by `mobile_no` (safer than a raw create).

Responses are predictable: a single doc is `{ "data": {...} }`, a list is `{ "data": [...] }`, a method returns `{ "message": <result> }`.

**Why it matters for you:** you rarely need a custom endpoint. Listing leads, reading one, fetching master data — it's all the resource API pointed at the right DocType. Reach for the method API only for `upsert_lead_by_phone` (the recommended way to write leads) and counts.

→ **Try it:** [List leads](/docs/api/leads#list-leads-filtered-paginated) · [Upsert a lead](/docs/api/leads#upsert-a-lead-by-phone-recommended).
