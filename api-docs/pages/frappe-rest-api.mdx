---
title: "The auto-generated REST API"
---

Because every record is a DocType, Frappe exposes **every DocType over REST automatically** — no endpoint is hand-written for CRUD. Two URL shapes cover almost everything.

**1. Resource API — `/api/resource/<DocType>`** (URL-encode the space, e.g. `CRM%20Lead`):

| Method | URL | Does |
|---|---|---|
| `GET` | `/api/resource/CRM Lead` | list (filter + paginate) |
| `GET` | `/api/resource/CRM Lead/{name}` | read one |
| `POST` | `/api/resource/CRM Lead` | create |
| `PUT` | `/api/resource/CRM Lead/{name}` | update |
| `DELETE` | `/api/resource/CRM Lead/{name}` | delete |

**2. Method API — `/api/method/<dotted.path>`** for custom server functions — e.g. the partner lead methods under `tatva_connect.api.partner.*` (`lead_create`, `lead_get`, `lead_list`, …), which find-or-create and read leads with the right scoping and merge behaviour.

Responses are predictable: a single doc is `{ "data": {...} }`, a list is `{ "data": [...] }`, a method returns `{ "message": <result> }`.

**Why it matters for you:** the resource API above is available to **internal System Manager keys** only. **Partner keys must use the method API** (`tatva_connect.api.partner.*`) — raw `/api/resource/CRM Lead` returns **403** for them. So for partners, the lead methods (`lead_create` to write, `lead_get` / `lead_list` to read) are the entire surface; the resource shapes are shown here so you understand the response envelope.
